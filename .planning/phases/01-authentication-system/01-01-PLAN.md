<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser to API | Future credentialed cookie requests cross from untrusted browser to trusted API. |
| API to PostgreSQL | Backend writes user/auth state to database service. |
| Developer env to secrets | `.env` values control JWT and database access. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-01 | Information Disclosure | `.env.example` | mitigate | Document names only, never real secrets; add placeholder values. |
| T-1-02 | Tampering | CORS setup | mitigate | Configure Nest CORS with explicit origin and `credentials: true`, not wildcard credentials. |
| T-1-03 | Denial of Service | Test harness | accept | Foundation tests are local-only and low risk. |
</threat_model>
