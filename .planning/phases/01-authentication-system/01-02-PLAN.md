<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser to auth API | Untrusted credentials and cookies enter API controllers. |
| API to database | Password hashes and token hashes are persisted. |
| API to cookie jar | Backend controls Set-Cookie headers consumed by browser. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-04 | Spoofing | JwtAuthGuard | mitigate | Verify JWT signature/exp and reject missing/malformed access cookies. |
| T-1-05 | Information Disclosure | Auth responses | mitigate | Never return passwordHash, refresh hash, or reset token values. |
| T-1-06 | Tampering | Refresh tokens | mitigate | Hash tokens, rotate on refresh, revoke on logout. |
| T-1-07 | Elevation of Privilege | Public routes | mitigate | Use explicit `@Public()` metadata and default-protect profile routes. |
| T-1-08 | Information Disclosure | Cookies | mitigate | Set httpOnly cookies per D-02; Secure in production; SameSite=Lax. |
</threat_model>
