<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Authenticated browser to profile API | User-controlled profile data enters protected API. |
| API to mail provider | Password reset links leave the system through SMTP. |
| Browser upload to API storage | Custom avatar file content enters server storage. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-1-09 | Elevation of Privilege | Profile API | mitigate | Scope all reads/writes to `CurrentUser.id`; ignore userId body fields. |
| T-1-10 | Information Disclosure | Password reset request | mitigate | Return same public response for known and unknown emails. |
| T-1-11 | Spoofing | Reset token | mitigate | Store reset token hash, expire token, and invalidate after use. |
| T-1-12 | Tampering | Avatar upload | mitigate | Validate MIME type/size and store generated filenames only. |
</threat_model>
